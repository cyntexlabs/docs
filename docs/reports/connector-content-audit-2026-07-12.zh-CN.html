结论

连接器文档迁移在“可公开、可操作、可由当前证据支撑”的范围内已经收口。

49 篇指南都已覆盖角色、读取模式、准备工作、创建连接、验证边界、限制和配置参考；所有 catalog 字段都能在对应页面找到。这里的“完整”不是逐字复制 docs-en，而是保留仍适用于 TapState 的客户信息，同时移除旧平台 UI、Agent、心跳表和无法由 TapState 证明的功能。

文档完整不等于 49 个连接器都已真实跑通。

当前开源产品的连接探针仍未实现，因此凭证、网络、权限、TLS、真实读写和恢复行为不能由本次文档构建证明。页面中的 Local validation boundary 已明确这个边界。

01 · TAPSTATE CATALOG

确认连接器 ID、Source 模式、Target 标记、字段拼写、默认值和枚举。

02 · CONNECTOR CODE

确认 spec、配置消费路径、解析器、CDC 依赖、数据类型和实现限制。

03 · DOCS-EN

补齐授权 SQL、服务端设置、兼容版本、注意事项和故障恢复步骤。

04 · RUNTIME

只有真实执行结果才能证明网络、认证、读写、TLS、恢复和容量表现。

有意不迁移的内容

下面不是内容遗漏，而是避免把旧产品行为或未经验证的能力写成 TapState 功能。

产品界面与控制面

  • TapData 左侧导航、Create/Test/Save 流程和截图
  • Agent 自动分配、模型刷新、心跳表和共享挖掘 UI
  • 旧平台的高级节点开关，除非 TapState schema 已暴露

尚无证据的运行能力

  • 未暴露字段的 TLS、证书上传和连接测试
  • 没有 TapState 测量环境的 RPS、吞吐量和推荐批量数
  • 只在旧 UI 出现的冲突策略、自动建表和 DDL 开关

连接器特例

  • Oracle direct raw-log 路径：需要独立组件，公开 TapState contract 未提供
  • Zoho Desk WebHook：上游存在，但 catalog 只声明 Snapshot
  • Paimon/StarRocks Source：catalog 声明，docs-en 主要写 Target，因此要求 staging 验证

迁移原则

  • 正文只保留客户需要执行或判断的内容
  • 来源 SHA、差异和证据等级放在本报告，而不是每页 frontmatter
  • 不维护第二套 machine-readable facts；MDX 正文同时服务人和 LLM
