TapState separates AI assistance from deterministic product control. An assistant can help discover documentation, draft resources, and interpret diagnostics; schema, connector contracts, runtime state, authorization, and human review remain authoritative.

## Documentation-assisted authoring

```text
Docs and page Markdown
          ↓
AI or human drafts resources
          ↓
Product schema and validation
          ↓
Reviewed files and diagnostics
```

Useful documentation surfaces include:

- `llms.txt` for discovery;
- page-level Markdown for canonical content;
- connector frontmatter for IDs, roles, modes, and aliases;
- resource reference for exact field lookup;
- coded diagnostics from an identified product artifact.

An assistant should never convert a successful syntax check into a claim that credentials, connectivity, or data movement succeeded.

## Runtime control direction

A future live control surface can expose discovery, apply, lifecycle, status, logs, and metrics through CLI, API, Web, or MCP clients. Those clients should call the same operation contracts and authorization layer rather than implement separate behavior.

<Aside type="note" title="Interface details require a release contract">
  MCP endpoints, tool names, authentication, and lifecycle commands are product facts. Treat them as design direction until an identified TapState release publishes and verifies them.
</Aside>

## Human authorization boundary

- Read operations must remain within granted scope.
- Generated writes must identify the resource and intended change.
- Destructive and lifecycle operations require explicit authorization.
- Credentials should stay outside generated resources where possible.
- An AI suggestion cannot override schema, Catalog, runtime, or access-control decisions.
- A human should review generated permissions and database commands before execution.

## Deterministic surfaces

| Need | Authoritative surface |
|---|---|
| Resource shape | Published schema and parser |
| Connector fields and modes | Current connector contract |
| Closed references | Workspace validation |
| Runtime condition | Deployment status and logs |
| Authorization | Product access-control decision |
| AI context | Canonical page body and versioned reference |

See [Authoring with AI](/docs/for-ai/authoring) for the documentation workflow and [Release status](/docs/overview/release-status) for the evidence boundary.
