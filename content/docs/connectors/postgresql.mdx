## Before you begin

### Source

<SourceModeTabs label="Choose the PostgreSQL source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to copy the rows that exist now.

1. Create a dedicated login with `CONNECT` access to the database.
2. Grant `USAGE` on selected schemas and `SELECT` on selected tables.
3. Grant access to future tables when table discovery must include objects created later.

Snapshot does not require logical WAL, a replication slot, or a decoding plugin.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** to copy existing rows and then follow WAL changes.

1. Grant the Snapshot permissions plus replication access.
2. Set `wal_level = logical` and size `max_replication_slots` and `max_wal_senders` for the planned tasks.
3. Choose the supported decoding path—`pgoutput`, `wal2json`, or `decoderbufs`—and install it when required.
4. Set `REPLICA IDENTITY` appropriately for tables whose updates or deletes must include enough key data.
5. For DDL capture, prepare the event-trigger and audit-table permissions described by the reused connector.

</SourceModeTab>
</SourceModeTabs>

### Target

Prepare a dedicated writer with schema usage, table creation when TapState may create targets, and the required `INSERT`, `UPDATE`, and `DELETE` privileges. Add DDL privileges only when DDL apply is allowed.

## Create a connection

Do not create a production `.cyn.yml` resource from this page yet. TapState must first publish the PostgreSQL config fields, modes, sink semantics, and supported logical-decoding choices in its catalog.

## Validate the configuration

The current offline CLI cannot validate a complete PostgreSQL connection contract. Validation becomes meaningful only after the catalog exposes its modes, conditional fields, and target capability.

## Limitations

- GA describes the reused connector's certification level, not current TapState runtime availability.
- Replication slots retain WAL; monitor disk growth and remove abandoned slots through an approved operational process.
- Tables without a suitable replica identity can produce incomplete update or delete keys.
- SSL and advanced node settings remain pending until the TapState catalog exposes their fields.
