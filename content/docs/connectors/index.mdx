## Maturity labels

- **GA** means the connector has a stable, generally available contract intended for production use.
- **Preview** means the connector is ready to evaluate, but its configuration or behavior may still change before GA. Validate production workloads with the connector contract supplied by your TapState deployment.
- **Deprecated** means the connector remains documented for existing deployments, but new work should use its named replacement.

TapState uses GA and Preview for current connectors. The upstream inventory's former **Beta** group maps to **Preview**, a clearer customer-facing term. **Deprecated** is retained only when a current catalog explicitly marks a legacy connector that customers may still need to maintain. We do not show **Alpha** until a connector has a documented, limited-access contract.

## How to read roles and modes

- **Source** reads from the listed system. The read mode tells you whether that happens as a one-time Snapshot, Snapshot + CDC, stream, or API read.
- **Target** means the current catalog declares data writing. Check the connector guide before relying on table creation, schema evolution, or a particular write behavior.
- **Not declared** means the catalog has a source connection contract but does not declare a readable mode. A dash means the guide is target-only. Neither value implies an undocumented source mode.

<Aside type="note" title="Validation boundary">
  `tapstate validate` checks the resource envelope plus catalog-known connector IDs, modes, and field shapes. It does not prove network reachability, credentials, permissions, API scope, data access, or a live write. Individual guides state the preparation and runtime checks required for that system.
</Aside>
