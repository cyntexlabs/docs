## How capability claims are made

| Signal | Meaning |
|---|---|
| Maturity | The connector lifecycle level, such as GA or Preview. |
| Source | The supported way to read data: Snapshot, CDC, stream, or API. |
| Target | Whether the current connector contract declares data writing. |
| Schema changes | DDL handling only when it is explicitly evidenced; otherwise it is not claimed. |

## Create and validate a connection

Connection resources are `.tapstate.yml` files. Use the exact connector ID and field names from the page's Reference section. A target is represented by the same `kind: source` connection resource without a reader `mode` or `tables` section.

```bash
tapstate validate --workdir tapstate-work
```

This offline check validates workspace structure plus catalog-known modes and field shapes. It does not test a network route, credentials, database privileges, API access, or a live write.
