## Before you begin

### Source

<SourceModeTabs>
<SourceModeTab value="snapshot">

Use **Snapshot** when you need the rows that exist now. Allow the future runtime network to reach the selected tenant, then create a least-privilege reader:

```sql
CREATE USER 'tapstate_reader' IDENTIFIED BY 'replace_with_a_secret';
GRANT SELECT ON `orders`.* TO 'tapstate_reader';
```

A one-time copy does not require the OceanBase log service.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** when the target must start current and then receive later data changes. In addition to the reader access, deploy the OceanBase Binlog Service described by the connector. It uses OBProxy for client access and `obbinlog` for incremental log conversion.

**1. Create a dedicated reader**

```sql
CREATE USER 'tapstate_reader' IDENTIFIED BY 'replace_with_a_secret';
GRANT SELECT ON `orders`.* TO 'tapstate_reader';
```

**2. Prepare the Binlog Service path**

Record the Root Server list, raw-log server host and port, plus CDC credentials required by your deployment. Configure them as `rootServerList`, `rawLogServerHost`, `rawLogServerPort`, `cdcUser`, and `cdcPassword`, then validate the service endpoint with the deployed runtime before relying on incremental delivery.

**3. Decide whether to use native CDC**

`useNativeCdc` defaults to `false`. When enabled, the connector uses its native CDC reader. Set it only when the deployed TapState runtime and OceanBase topology have been validated for that path.

</SourceModeTab>
</SourceModeTabs>

### Target

Create a dedicated writer for the destination database. Grant schema-management privileges only when a pipeline is explicitly allowed to create or alter destination tables.

```sql
CREATE USER 'tapstate_writer' IDENTIFIED BY 'replace_with_a_secret';
GRANT ALL ON `analytics`.* TO 'tapstate_writer';
```

Scope that grant more narrowly where the tenant's authorization model allows it.

## Create a connection

```yaml title="tapstate-work/source/orders_oceanbase.tapstate.yml"
version: tapstate/v1
kind: source
id: orders_oceanbase
connector: oceanbase
config:
  host: obproxy.internal
  port: "2881"
  tenant: sales
  database: orders
  user: ${OCEANBASE_READER_USER}
  password: ${OCEANBASE_READER_PASSWORD}
  rootServerList: ob-root-1.internal:2882,ob-root-2.internal:2882
  rawLogServerHost: oblog.internal
  rawLogServerPort: "50051"
  cdcUser: root@sys
  cdcPassword: ${OCEANBASE_CDC_PASSWORD}
  useNativeCdc: false
  timezone: "+00:00"
mode: cdc
tables:
  - orders
```

Use `mode: snapshot` for a one-time copy. For a target connection, omit `mode` and `tables`, and use the writer credentials. CDC-specific values are optional; set them only for the deployed service topology that needs them.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

Local validation checks the resource, selected mode, and configuration shape. It does not reach OBProxy, check a tenant allowlist, verify Binlog Service deployment, authenticate CDC credentials, or write destination data.

## Limitations

- This page covers MySQL-compatible OceanBase only. Oracle-compatible OceanBase needs its own guide.
- Source DDL capture is not documented. The target connector implements add column, rename column, alter column attributes, and drop column; stage each operation before using `ddl: apply`.
- Treat tenant access, cluster topology, Binlog Service setup, and type compatibility as deployment-specific. Test them with representative data before production use.

## Reference

| Field | Required | Default | Use |
|---|---|---|---|
| `host` | Yes | — | OceanBase endpoint, commonly an OBProxy host. |
| `port` | Yes | `2881` | OceanBase service port; provide it as a string. |
| `tenant` | No | — | OceanBase logical tenant. |
| `database` | Yes | — | Database name. |
| `user`, `password` | `user` yes | — | Tenant account and optional password; inject the password through an environment variable. |
| `extParams` | No | — | Connector-specific connection parameters. |
| `rootServerList` | No | — | OceanBase Root Server list for a CDC deployment. |
| `rawLogServerHost`, `rawLogServerPort` | No | `50051` port | Raw log service endpoint. |
| `cdcUser`, `cdcPassword` | No | `root@sys` user | CDC service credentials. |
| `useNativeCdc` | No | `false` | Selects the connector's native CDC reader when the deployed runtime supports it. |
| `timezone` | No | empty | UTC offset used by the connector. |

### Data compatibility

| Family | Supported baseline |
|---|---|
| Text and binary strings | `CHAR`, `VARCHAR`, `TINYTEXT`, `TEXT`, `MEDIUMTEXT`, `LONGTEXT`, `BINARY`, `VARBINARY` |
| Integer | `TINYINT`, `SMALLINT`, `MEDIUMINT`, `INT`, `BIGINT` |
| Numeric | `DECIMAL`, `FLOAT`, `DOUBLE` |
| Date and time | `DATE`, `TIME`, `DATETIME`, `TIMESTAMP`, `YEAR` |
| Enumerated and structured | `ENUM`, `SET`, `JSON`, `BIT` |
| Binary large object | `TINYBLOB`, `BLOB`, `MEDIUMBLOB`, `LONGBLOB` |
| Spatial | `POINT`, `LINESTRING`, `POLYGON`, `GEOMETRY`, `MULTIPOINT`, `MULTILINESTRING`, `MULTIPOLYGON`, `GEOMCOLLECTION` |
