## Before you begin

### Source

Create a least-privilege reader for the selected database:

```sql
CREATE USER tapstate_reader IDENTIFIED WITH sha256_password BY 'replace_with_a_secret';
GRANT SELECT ON analytics.* TO tapstate_reader;
```

### Target

Create a writer. Add table-creation and alteration permissions only when pipelines are allowed to manage destination tables.

```sql
CREATE USER tapstate_writer IDENTIFIED WITH sha256_password BY 'replace_with_a_secret';
GRANT SELECT, INSERT, CREATE TABLE, ALTER TABLE,
      ALTER UPDATE, DROP TABLE, TRUNCATE
ON analytics.* TO tapstate_writer;
```

Remove creation, alteration, drop, and truncate privileges when the pipeline writes only to existing tables.

## Create a connection

```yaml title="tapstate-work/source/analytics_clickhouse.tapstate.yml"
version: tapstate/v1
kind: source
id: analytics_clickhouse
connector: clickhouse
config:
  host: clickhouse.internal
  port: "8123"
  database: analytics
  user: ${CLICKHOUSE_USER}
  password: ${CLICKHOUSE_PASSWORD}
mode: snapshot
tables:
  - events
```

For a target-only connection, omit `mode` and `tables`.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

This checks the local resource, Snapshot mode, and catalog-known field types—not network access, ClickHouse privileges, or query results.

## Limitations

- The catalog declares Snapshot only; it does not declare a CDC or DDL contract.
- ClickHouse HTTP is commonly served on port `8123`; confirm the endpoint and any encryption settings with your deployment rather than assuming a TLS field.
- The connector implementation creates `ReplacingMergeTree` with the primary key as the sort key when the source has a primary key; otherwise it creates `MergeTree` without a sort key. Pre-create tables when that default does not fit the query or retention design.
- Binary source values are Base64-encoded into ClickHouse strings by the connector implementation because the documented ClickHouse type baseline has no binary type.
- Updates and deletes on MergeTree-family tables are operationally different from row-store DML. Test table engines, sort keys, mutation behavior, and deduplication with representative data before production writes.

## Reference

| Field | Required | Default | Use |
|---|---|---|---|
| `host` | Yes | — | ClickHouse host. |
| `port` | Yes | `8123` | HTTP endpoint port. |
| `database` | Yes | — | Database name. |
| `user` | Yes | — | Database user. |
| `password` | No | — | Password; inject through an environment variable. |
| `extParams` | No | — | Connector-specific connection parameters. |
| `timezone` | No | empty | UTC offset used by the connector. |
