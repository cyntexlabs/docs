## Before you begin

### Source

Create a dedicated reader and grant access only to the schema and tables that TapState will copy.

```sql
CREATE USER tapstate_reader PASSWORD 'replace_with_a_secret';
GRANT SELECT ON ALL TABLES IN SCHEMA public TO tapstate_reader;
```

Run future-table grants through your approved database administration process when newly created tables must be included.

### Target

Create a dedicated writer for the destination schema.

```sql
CREATE USER tapstate_writer PASSWORD 'replace_with_a_secret';
GRANT INSERT, UPDATE, DELETE, TRUNCATE ON ALL TABLES IN SCHEMA public TO tapstate_writer;
```

Add object-creation or alteration privileges only when your pipeline is explicitly allowed to create or evolve destination tables.

## Create a connection

### Snapshot source

```bash
tapstate new --non-interactive \
  --kind source \
  --connector open-gauss \
  --id orders_opengauss \
  --mode snapshot \
  --set host=opengauss.internal \
  --set port=5432 \
  --set database=orders \
  --set schema=public \
  --set user='${OPENGAUSS_READER}' \
  --set password='${OPENGAUSS_READER_PASSWORD}' \
  --workdir tapstate-work
```

The resulting source connection uses catalog-backed field names:

```yaml title="tapstate-work/source/orders_opengauss.tapstate.yml"
version: tapstate/v1
kind: source
id: orders_opengauss
connector: open-gauss
config:
  host: opengauss.internal
  port: "5432"
  database: orders
  schema: public
  user: ${OPENGAUSS_READER}
  password: ${OPENGAUSS_READER_PASSWORD}
  logPluginName: pgoutput
mode: snapshot
tables:
  - orders
```

### Target connection

Targets use the same connection-resource envelope, without a read mode or table selection.

```yaml title="tapstate-work/source/analytics_opengauss.tapstate.yml"
version: tapstate/v1
kind: source
id: analytics_opengauss
connector: open-gauss
config:
  host: opengauss.internal
  port: "5432"
  database: analytics
  schema: public
  user: ${OPENGAUSS_WRITER}
  password: ${OPENGAUSS_WRITER_PASSWORD}
  logPluginName: pgoutput
```

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

The command validates the local resource structure, `snapshot` mode, and known catalog field shapes. It does not validate a network route, database credentials, permissions, or a live write.

## Limitations

- The current catalog declares Snapshot, not CDC. Do not configure `mode: cdc` for openGauss from this page.
- The `logPluginName` setting is catalog-backed, but its presence does not prove a supported CDC path.
- Connection encryption and advanced runtime options are deployment-specific; this page does not infer them from upstream UI documentation.

## Reference

| Field | Required | Default | Use |
|---|---|---|---|
| `host` | Yes | — | Database host. |
| `port` | Yes | — | Database listener port. |
| `database` | Yes | — | Database name. |
| `schema` | Yes | — | Schema to read from or write to. |
| `user` | No | — | Database user. Prefer an environment variable. |
| `password` | No | — | Database password. Prefer an environment variable. |
| `extParams` | No | — | Connector-specific connection parameters. |
| `logPluginName` | Yes | `pgoutput` | Catalog enum: `pgoutput`, `decoderbufs`, and listed `wal2json` variants. |
| `timezone` | No | empty | UTC offset used by the connector. |
