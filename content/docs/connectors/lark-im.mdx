<Aside type="note" title="Local validation boundary">
  Local validation checks the resource and application credential field shapes. It does not authenticate to Lark, verify app publication, resolve recipients, or send a message.
</Aside>

## Before you begin

### Target

1. Create or select a Lark/Feishu self-built application.
2. Copy its App ID and App Secret into a secret manager.
3. Set the application's availability scope so intended users and chats are visible to the app.
4. Enable the bot capability and publish the app before production delivery.
5. Design downstream delivery within Lark limits: no more than 5 messages per second per recipient or group, and keep message bodies within the platform limits.

The message payload needs a recipient type and ID plus Lark message content. A typical record has this shape:

```json
{
  "receiveType": "chat",
  "receiveId": "oc_xxxxxxxxx",
  "contentType": "text",
  "content": "{\"text\":\"Order 42 needs review\"}"
}
```

## Create a connection

```yaml title="tapstate-work/target/notifications_lark.tapstate.yml"
version: tapstate/v1
kind: target
id: notifications_lark
connector: lark-im
config:
  app_id: ${LARK_APP_ID}
  app_secret: ${LARK_APP_SECRET}
  app_name: operations-notifier
```

Target connections do not have a source `mode` or `tables` section. Keep `app_secret` outside source control.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

Before production, send a representative text and interactive-message payload to a test recipient in the app's visibility scope.

## Limitations

- Recipient visibility, bot publication, rate limits, and message-format validation are enforced by Lark/Feishu at runtime.
- The connector is for message delivery; it does not provide a documented source read mode.
- Transform upstream records into the recipient and content shape expected by Lark before delivery.

## Reference

| Field | Required | Use |
|---|---|---|
| `app_id` | Yes | Self-built application App ID. |
| `app_secret` | Yes | Application secret; inject it through an environment variable. |
| `app_name` | No | Optional application name for connection metadata. |
