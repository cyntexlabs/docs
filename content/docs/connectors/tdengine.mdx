## Before you begin

### Source

<SourceModeTabs>
<SourceModeTab value="snapshot">

Use **Snapshot** to copy current data once. The upstream baseline uses TDengine's REST endpoint on port `6041`; allow the future runtime to reach that endpoint and create a read account for the selected database.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** when you also need later records. The upstream connector baseline describes TDengine new-data subscriptions, port `6030`, and a TDengine client driver. The catalog exposes `originPort` with default `6030`; confirm the deployed runtime's driver and subscription requirements before relying on this path.

</SourceModeTab>
</SourceModeTabs>

### Target

Create a dedicated writer for the destination database. Decide whether the pipeline may create normal, super, or child tables before granting schema-management privileges.

## Create a connection

```yaml title="tapstate-work/source/telemetry_tdengine.tapstate.yml"
version: tapstate/v1
kind: source
id: telemetry_tdengine
connector: tdengine
config:
  host: tdengine.internal
  port: "6041"
  database: telemetry
  user: ${TDENGINE_USER}
  password: ${TDENGINE_PASSWORD}
  loadTableOptions: [NORMAL_TABLE, SUPER_TABLE, CHILD_TABLE]
  originPort: "6030"
mode: cdc
tables:
  - readings
```

Use `mode: snapshot` for a one-time copy. For a target connection, omit `mode` and `tables`, then use destination credentials.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

Local validation checks the resource envelope, mode, and catalog-known fields. It does not test REST access, the native subscription port, a driver installation, permissions, or record delivery.

## Limitations

- The catalog declares Snapshot and CDC but no DDL contract; this guide does not claim DDL capture or application.
- The TDengine 3.x, REST-port, and subscription-driver statements are an upstream compatibility baseline, not a TapState runtime test.
- The upstream baseline notes that TDengine cannot change the width of some column types. Test table and type behavior with representative data before production writes.
