## Before you begin

### Source

<SourceModeTabs>
<SourceModeTab value="snapshot">

Use **Snapshot** to copy current data once. The connector uses TDengine's REST endpoint on port `6041`; allow the future runtime to reach that endpoint and create a read account for the selected database.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** when you also need later records. The connector describes TDengine new-data subscriptions, port `6030`, and a TDengine client driver. The connection exposes `originPort` with default `6030`; confirm the deployed runtime's driver and subscription requirements before relying on this path.

</SourceModeTab>
</SourceModeTabs>

### Target

Create a dedicated writer for the destination database. Decide whether the pipeline may create normal, super, or child tables before granting schema-management privileges.

## Create a connection

```yaml title="tapstate-work/source/telemetry_tdengine.tapstate.yml"
version: tapstate/v1
kind: source
id: telemetry_tdengine
connector: tdengine
config:
  host: tdengine.internal
  port: "6041"
  database: telemetry
  user: ${TDENGINE_USER}
  password: ${TDENGINE_PASSWORD}
  loadTableOptions: [NORMAL_TABLE, SUPER_TABLE, CHILD_TABLE]
  originPort: "6030"
mode: cdc
tables:
  - readings
```

Use `mode: snapshot` for a one-time copy. For a target connection, omit `mode` and `tables`, then use destination credentials.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

Local validation checks the resource envelope, mode, and recognized fields. It does not test REST access, the native subscription port, a driver installation, permissions, or record delivery.

## Limitations

- TapState supports Snapshot and CDC but does not document source DDL capture. The target connector implements add column, rename column, alter column attributes, and drop column through the pipeline DDL policy.
- The TDengine 3.x, REST-port, and subscription-driver statements are an documented compatibility, not a TapState runtime test.
- The connector notes that TDengine cannot change the width of some column types. Test table and type behavior with representative data before production writes.

|---|---|---|---|
| `host` | Yes | — | TDengine host. |
| `port` | Yes | `6041` | REST endpoint port. |
| `database` | Yes | — | Database name. |
