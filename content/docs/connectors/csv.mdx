## Before you begin

### Source

<SourceModeTabs label="Choose the CSV source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to read the files that match the configured path now. Place files where the TapState runtime can reach them. For remote storage, use a dedicated account restricted to the required path or bucket. Keep headers and delimiters consistent across files in the same logical dataset.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** only after staging the deployed file-change behavior. TapState supports `cdc`, but does not define the watcher, polling interval, changed-file policy, or deletion behavior. Confirm those semantics with a controlled add, replace, rename, and delete in the actual storage system.

</SourceModeTab>
</SourceModeTabs>

### Target

Choose an output path and storage account with write access limited to that location. Before production, agree on the naming, overwrite, partitioning, and conflict policy with downstream file consumers.

## Create a connection

```yaml title="tapstate-work/source/orders_csv.tapstate.yml"
version: tapstate/v1
kind: source
id: orders_csv
connector: csv
config:
  protocol: sftp
  encoding: UTF-8
  sftpHost: sftp.internal
  sftpPort: "22"
  sftpUsername: ${SFTP_USER}
  sftpPassword: ${SFTP_PASSWORD}
  filePathString: /exports/orders/*.csv
mode: snapshot
```

For a target connection, omit `mode`; validate naming, overwrite, and write semantics with the runtime before production use.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- The connection exposes Snapshot, CDC, and target roles but does not define a file-change mechanism, deletion behavior, output layout, or write conflict policy.
- The connector describes file polling and source types, but that behavior is not published as a TapState runtime guarantee.
- Treat `oss` as a storage protocol option, not an endorsement of a regional deployment; S3-compatible storage is also available through `s3fs`.

## Reference

| Scope | Fields | Notes |
|---|---|---|
| Common | `protocol`, `encoding`, `filePathString` | Protocol defaults to `local`; encoding defaults to `UTF-8`. |
| FTP | `ftpHost`, `ftpPort`, `ftpUsername`, `ftpPassword`, `ftpPassiveMode`, `ftpConnectTimeout`, `ftpDataTimeout` | Defaults: port `21`, passive mode `true`, connect timeout `60`, data timeout `0`. |
| SFTP | `sftpHost`, `sftpPort`, `sftpUsername`, `sftpPassword` | Default SFTP port is `22`. |
| SMB | `smbHost`, `smbUsername`, `smbPassword`, `smbShareDir` | Select this group only for `smb`. |
| S3FS / OSS | `accessKey`, `secretKey`, `endpoint`, `region`, `bucket` | Use secret-aware variables for key material. |

`protocol` accepts `local`, `ftp`, `sftp`, `smb`, `s3fs`, or `oss`. The connection schema marks transport-specific fields as required; provide only the fields relevant to the selected protocol and verify the deployed connector's conditional validation.
