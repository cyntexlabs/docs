<Aside type="note" title="Local validation boundary">
  The catalog validates Stream mode and the listed connection fields. It does not open a broker connection, verify credentials, list queues, consume a message, or publish one.
</Aside>

## Before you begin

### Source

Create a broker account restricted to consuming from the intended queues. The reused connector accepts JSON-object messages and queue mode; test message shape and acknowledgement behavior with the deployed runtime.

### Target

Grant the same account permission to publish only to the intended queues. Pre-create production queues with the retention, durability, and dead-letter policy required by downstream consumers.

## Create a connection

```yaml title="tapstate-work/source/events_activemq.tapstate.yml"
version: tapstate/v1
kind: source
id: events_activemq
connector: activemq
config:
  mqQueueString: orders.events
  brokerURL: tcp://activemq.internal:61616
  mqUsername: ${ACTIVEMQ_USER}
  mqPassword: ${ACTIVEMQ_PASSWORD}
mode: stream
```

For a target-only connection, omit `mode`.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- The catalog does not define delivery guarantee, acknowledgement, transactions, topic support, CDC, or DDL behavior.
- The documented compatibility is not a deployed TapState runtime test; stage Classic and Artemis broker URLs separately.

## Reference

| Field | Required | Use |
|---|---|---|
| `mqQueueString` | No | Optional queue selection. |
| `brokerURL` | Yes | Broker connection string, for example `tcp://host:61616`. |
| `mqUsername`, `mqPassword` | No | Broker credentials. Keep the password in an environment variable. |

### Message data compatibility

The reused connector baseline accepts `OBJECT`, `ARRAY`, `NUMBER`, `INTEGER`, `BOOLEAN`, `STRING` values shorter than 200 characters, and `TEXT`. Verify the actual message envelope and character encoding with a staging queue.
