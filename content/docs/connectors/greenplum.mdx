## Before you begin

### Source

Create a reader account limited to the selected schema. Although the connection includes `logPluginName`, TapState currently publishes Snapshot only for Greenplum; do not configure an incremental path based on that field alone.

### Target

Create a separate writer account limited to the destination schema. Grant DDL privileges only when the pipeline is approved to manage destination tables.

## Create a connection

```yaml title="tapstate-work/source/mart_greenplum.tapstate.yml"
version: tapstate/v1
kind: source
id: mart_greenplum
connector: greenplum
config:
  host: greenplum.internal
  port: "5432"
  database: analytics
  schema: public
  user: ${GREENPLUM_USER}
  password: ${GREENPLUM_PASSWORD}
  logPluginName: pgoutput
mode: snapshot
tables:
  - orders
```

For a target connection, omit `mode` and `tables`.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- `logPluginName` is required by the connection schema but does not establish CDC support; TapState supports Snapshot only.
- Target-side add column, rename column, alter column attributes, and drop column are implemented by the connector. Stage each operation before changing a pipeline from `ddl: fail` to `ddl: apply`.
- This guide does not define topology/failover semantics or TLS fields.

## Reference

| Field | Required | Default | Use |
|---|---|---|---|
| `host`, `port`, `database`, `schema` | Yes | — | Greenplum endpoint and selected schema. |
| `user`, `password` | No | — | Connection account and secret. |
| `logPluginName` | Yes | `pgoutput` | Values: `pgoutput`, `decoderbufs`, `wal2json`, `wal2json_rds`, `wal2json_streaming`, and `wal2json_rds_streaming`. This Snapshot-only guide does not establish a CDC path. |
| `extParams`, `timezone` | No | empty timezone | Extra parameters and UTC offset. |
