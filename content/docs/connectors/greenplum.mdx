## Before you begin

Create separate reader and writer accounts limited to the selected schema. The upstream guide exposes a `logPluginName` setting for incremental work, but the current catalog declares Snapshot only; do not configure an incremental path based on that field alone.

## Create a connection

```yaml title="tapstate-work/source/mart_greenplum.tapstate.yml"
version: tapstate/v1
kind: source
id: mart_greenplum
connector: greenplum
config:
  host: greenplum.internal
  port: "5432"
  database: analytics
  schema: public
  user: ${GREENPLUM_USER}
  password: ${GREENPLUM_PASSWORD}
  logPluginName: pgoutput
mode: snapshot
tables:
  - orders
```

For a target connection, omit `mode` and `tables`.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- `logPluginName` is required by the catalog but does not establish CDC support; the catalog declares Snapshot only.
- The catalog does not declare DDL behavior, topology/failover semantics, or TLS fields.

## Reference

| Field | Required | Default | Use |
|---|---|---|---|
| `host`, `port`, `database`, `schema` | Yes | — | Greenplum endpoint and selected schema. |
| `user`, `password` | No | — | Connection account and secret. |
| `logPluginName` | Yes | `pgoutput` | Plugin enum includes `pgoutput`, `wal2json`, and related variants. |
| `extParams`, `timezone` | No | empty timezone | Extra parameters and UTC offset. |
