## Before you begin

### Source

<SourceModeTabs label="Choose the SQL Server source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to copy the rows that exist now.

1. Create a dedicated SQL Server login and database user.
2. Grant `CONNECT` and read access to the selected schemas or tables.
3. Ensure the future TapState runtime can reach the SQL Server listener and database.

Snapshot does not require SQL Server Agent or database/table CDC.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** to copy existing rows and then follow SQL Server CDC changes.

1. Grant the Snapshot permissions plus access to the CDC metadata and change tables required by the reused connector.
2. Ensure SQL Server Agent is running.
3. Enable CDC on the database and every participating table.
4. Set CDC cleanup retention longer than the longest expected interruption.
5. Prepare DDL-capture permissions only when schema changes must be collected.

</SourceModeTab>
</SourceModeTabs>

### Target

Prepare a dedicated writer with database access and the required `INSERT`, `UPDATE`, and `DELETE` privileges. Add object-creation and alteration privileges only when TapState is allowed to create or evolve target tables.

## Create a connection

Create the SQL Server connection through the connection flow supplied by your TapState deployment. The public offline catalog does not contain a SQL Server field schema, so this page does not publish a speculative `.tapstate.yml` example.

## Validate the configuration

The bundled offline CLI cannot validate SQL Server field names, modes, SQL Server Agent, database/table CDC, credentials, or retention. Use your deployment's connection validation and a staging run to confirm them.

## Limitations

- CDC must be enabled and retained for every participating table; expired change-table positions require a new snapshot.
- SQL Server edition and deployment constraints must be verified against the eventual TapState runtime.
- Connection encryption settings depend on the SQL Server connector contract supplied by your deployment; this page does not assume a field name or TLS behavior.
