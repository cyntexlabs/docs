## Before you begin

### Source

<SourceModeTabs label="Choose the Excel source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to read the matching workbooks once. Place them where the TapState runtime can reach them, then test the intended sheet structure with a representative file. The connector accepts XLS and XLSX workbooks, including merged cells and formulas, but that is not a substitute for a TapState runtime test.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** only after verifying file-change behavior with the deployed runtime. TapState supports `cdc`, but does not specify which workbook changes are detected, how rows are re-read, or how a deletion is represented.

</SourceModeTab>
</SourceModeTabs>

## Create a connection

```yaml title="tapstate-work/source/monthly_excel.tapstate.yml"
version: tapstate/v1
kind: source
id: monthly_excel
connector: excel
config:
  protocol: local
  encoding: UTF-8
  filePathString: /data/import/monthly-report.xlsx
mode: snapshot
```

For SFTP, SMB, S3FS, or OSS, add only the matching transport fields described in Reference.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

## Limitations

- TapState supports a source only, without target writes or DDL behavior.
- It does not define workbook-size limits, sheet selection, formula evaluation, file-change handling, or deletion behavior.
- Keep credentials outside committed workspace files and test memory use with production-sized workbooks.

## Reference

| Protocol | Fields |
|---|---|
| Common | `protocol`, `encoding`, `filePathString` |
| FTP | `ftpHost`, `ftpPort`, `ftpUsername`, `ftpPassword`, `ftpPassiveMode`, `ftpConnectTimeout`, `ftpDataTimeout` |
| SFTP | `sftpHost`, `sftpPort`, `sftpUsername`, `sftpPassword` |
| SMB | `smbHost`, `smbUsername`, `smbPassword`, `smbShareDir` |
| S3FS / OSS | `accessKey`, `secretKey`, `endpoint`, `region`, `bucket` |

`protocol` accepts `local`, `ftp`, `sftp`, `smb`, `s3fs`, and `oss`; `encoding` defaults to `UTF-8`.
