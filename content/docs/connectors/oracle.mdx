## Before you begin

### Source

<SourceModeTabs label="Choose the Oracle source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to copy the rows that exist now.

1. Create a dedicated Oracle account and grant session access.
2. Grant dictionary and object-read permissions required for the selected schemas and tables.
3. In multitenant environments, create the account and grants in the correct CDB or PDB context.

Snapshot does not require ARCHIVELOG, supplemental logging, or LogMiner.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** to copy existing rows and then capture redo-log changes.

1. Grant the Snapshot permissions plus the confirmed LogMiner and transaction-log access required by the reused connector.
2. Enable `ARCHIVELOG` and database-level supplemental logging.
3. Enable supplemental logging for participating tables and verify that archived logs cover the recovery window.
4. Grant `LOGMINING` on Oracle 12c and later when using LogMiner.
5. Prepare DDL-capture permissions only when DDL collection is required.

The upstream connector also supports a raw-log path, but TapState has not exposed a CDC-method selector, so this page does not recommend one.

</SourceModeTab>
</SourceModeTabs>

### Target

Prepare a dedicated writer with session, object creation, and DML privileges for the destination schema. Add `ALTER` and other DDL privileges only when DDL apply is explicitly allowed.

## Create a connection

Do not create a production `.cyn.yml` resource from this page yet. The catalog must first expose Oracle connection fields, multitenant settings, the supported CDC method, modes, and target semantics.

## Validate the configuration

The current offline CLI cannot validate an Oracle connection because the connector is absent from the TapState catalog. It also cannot test Oracle credentials, logging, supplemental logging, or LogMiner access.

## Limitations

- GA describes the reused connector's certification level, not current TapState runtime availability.
- LogMiner and raw-log capture have different type, architecture, and operational constraints.
- Oracle empty strings become `NULL`; heterogeneous targets and constraints require staging tests.
- SSL and advanced node settings remain pending until the TapState catalog exposes their fields.
