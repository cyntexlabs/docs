<Aside type="note" title="Authorization">
  The connection requires `code`. Obtain it through the authorization flow supplied by your TapState deployment; local validation does not perform the exchange.
</Aside>

## Before you begin

### Source

Have a Zoho CRM administrator approve the connected account and least-privilege CRM module access. Keep the authorization value in a secret manager or environment variable. Use the [Zoho API Console](https://api-console.zoho.com/) and the authorization flow supplied by your TapState environment to obtain the required code.

## Create a connection

```yaml title="tapstate-work/source/sales_zoho.tapstate.yml"
version: tapstate/v1
kind: source
id: sales_zoho
connector: zoho-crm
config:
  code: ${ZOHO_CRM_AUTHORIZATION_CODE}
mode: api
```

The current connection contract does not disclose token refresh, selected modules, or incremental behavior; confirm those with a deployed runtime.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

Local validation does not authorize Zoho, test permissions, enumerate modules, or read records.

## Limitations

- API source only; no target, Snapshot, CDC, DDL, or rate-limit contract is declared.
- The resource schema does not guarantee access to every Zoho CRM module; application scopes and account permissions still apply.
