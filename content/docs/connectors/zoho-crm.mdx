## Before you begin

Have a Zoho CRM administrator approve the connected account and least-privilege CRM module access. Keep the authorization value in a secret manager or environment variable.

## Create a connection

```yaml title="tapstate-work/source/sales_zoho.tapstate.yml"
version: tapstate/v1
kind: source
id: sales_zoho
connector: zoho-crm
config:
  code: ${ZOHO_CRM_AUTHORIZATION_CODE}
mode: api
```

The current catalog does not disclose token refresh, selected modules, or incremental behavior; confirm those with a deployed runtime.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

Local validation does not authorize Zoho, test permissions, enumerate modules, or read records.

## Limitations

- API source only; no target, Snapshot, CDC, DDL, or rate-limit contract is declared.
- The resource schema does not establish access to every upstream-listed CRM module.
