## Before you begin

### Source

Confirm the key namespace, value encoding, and account access needed for the data you intend to read. The catalog does not expose a Redis source-selection or CDC-position setting, so this page does not claim a specific Redis change-capture mechanism.

### Target

Choose the deployment mode‚Äî`standalone`, `sentinel`, or `cluster`‚Äîand create an account with the least Redis ACL permissions required by the destination keyspace. Decide how pipeline records map to keys and whether the schema metadata key should be retained before writing production data.

## Create a connection

```yaml title="tapstate-work/source/cache_redis.tapstate.yml"
version: tapstate/v1
kind: source
id: cache_redis
connector: redis
config:
  deploymentMode: standalone
  host: redis.internal
  port: "6379"
  sentinelName: ${REDIS_SENTINEL_NAME}
  database: "0"
  username: ${REDIS_USER}
  password: ${REDIS_PASSWORD}
  schemaKey: "-schema-key-"
mode: snapshot
```

Use `mode: cdc` only after staging it. For Sentinel, set `deploymentMode: sentinel` and provide `sentinelAddress`; for cluster mode, validate the exact endpoint layout with the deployed runtime. The catalog marks `sentinelName` as required even though it exposes all three deployment modes, so retain the field and confirm its expected value with your runtime contract.

For a target-only connection, omit `mode`.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

The local check validates field shape and the selected mode. It does not authenticate to Redis, discover cluster members, validate an ACL, or read/write keys.

## Limitations

- The catalog declares Snapshot and CDC but does not disclose the source key model or CDC mechanism; do not assume Redis Streams, keyspace notifications, or replication-log behavior.
- The upstream guide documents a target baseline for Redis 2.8‚Äì6.0. Test modern Redis versions and managed offerings before production use.
- The catalog does not declare DDL behavior or transport-encryption fields.
