## Before you begin

### Source

<SourceModeTabs label="Choose the TiDB source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to copy the rows that exist now. Full reads use the TiDB SQL endpoint and support the upstream version baseline beyond the narrower CDC range.

```sql
CREATE USER 'tapstate_reader'@'%' IDENTIFIED BY 'replace_with_a_secret';
GRANT SELECT ON orders.* TO 'tapstate_reader'@'%';
```

Allow the TapState runtime to reach the TiDB SQL service, normally port `4000`. Snapshot does not require PD or TiKV access.

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** for TiDB 6.0.0–8.1.9 when existing rows must be followed by ordered row changes. The reused connector integrates TiFlow/TiCDC 8.1.0.

1. Create the reader and grant `SELECT` as shown in the Snapshot tab.
2. Make PD available to the runtime on port `2379` and TiKV on port `20160`; keep TiDB, PD, TiKV, and the runtime on routable networks.
3. Ensure every selected table has a primary key or a unique index whose columns are non-null and not virtual.
4. Keep TiDB GC history longer than the longest expected interruption. The reused connector checks `tidb_gc_life_time` and recommends 24 hours as a starting point:

```sql
SET GLOBAL tidb_gc_life_time = '24h';
SHOW GLOBAL VARIABLES LIKE 'tidb_gc_life_time';
```

5. Confirm that the runtime host architecture is supported by the TiCDC binary supplied with the connector (`amd64` or `arm64` in the upstream baseline).

Do not substitute a self-compiled TiCDC binary in a production deployment without compatibility and recovery testing; that changes the connector runtime contract.

</SourceModeTab>
</SourceModeTabs>

### Target

Create a writer with DML privileges on destination tables:

```sql
CREATE USER 'tapstate_writer'@'%' IDENTIFIED BY 'replace_with_a_secret';
GRANT SELECT, INSERT, UPDATE, DELETE, ALTER, CREATE, DROP
ON analytics.* TO 'tapstate_writer'@'%';
```

Remove `ALTER`, `CREATE`, and `DROP` when pipeline-managed tables are not approved.

## Create a connection

```yaml title="tapstate-work/source/orders_tidb.tapstate.yml"
version: tapstate/v1
kind: source
id: orders_tidb
connector: tidb
config:
  host: tidb.internal
  port: "4000"
  database: orders
  user: ${TIDB_USER}
  password: ${TIDB_PASSWORD}
  pdServer: pd.internal:2379
  tiKvAddress: tikv.internal
  tiKvPort: "20160"
  timezone: "+00:00"
mode: cdc
tables:
  - orders
```

Use `mode: snapshot` for a one-time read. For target-only use, omit `mode` and `tables`.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- The catalog declares CDC but does not define its checkpoint, topology, DDL, or failover semantics.
- Full reads support all versions in the upstream baseline, but integrated TiCDC is documented for TiDB 6.0.0–8.1.9. Validate the exact TiDB, PD, and TiKV topology in staging.
- Tables without a non-null primary or unique key do not meet the documented TiCDC row-identity requirement.
- TiDB GC can remove history needed for restart. Choose `tidb_gc_life_time` from your recovery objective and monitor its operational cost.
- The catalog does not expose transport-encryption settings.

## Reference

| Field | Required | Default | Use |
|---|---|---|---|
| `host` | Yes | — | TiDB SQL host. |
| `port` | Yes | `4000` | TiDB SQL port; catalog type is string. |
| `database` | Yes | — | Database name. |
| `user`, `password` | No | — | Connection account and secret. |
| `pdServer` | No | — | PD endpoint exposed by the catalog. |
| `tiKvAddress`, `tiKvPort` | No | `20160` for port | TiKV endpoint fields exposed by the catalog. |
| `extParams` | No | — | Additional connector parameters. |
| `timezone` | No | `+00:00` | UTC offset for timezone-naive values. |

### Data compatibility

| Family | Supported baseline |
|---|---|
| Integer | Signed and unsigned `TINYINT`, `SMALLINT`, `MEDIUMINT`, `INT`/`INTEGER`, `BIGINT` |
| Text | `CHAR`, `VARCHAR`, `TEXT`, `TINYTEXT`, `MEDIUMTEXT`, `LONGTEXT`, `CLOB`, `ENUM`, `SET` |
| Numeric | Signed and unsigned `DECIMAL`, `FLOAT`, `DOUBLE`, plus `REAL` except in full-read baseline |
| Binary | `BINARY`, `VARBINARY`, `BLOB`, `TINYBLOB`, `MEDIUMBLOB`, `LONGBLOB` |
| Boolean | `BOOLEAN`, `BIT` |
| Date and time | `DATE`, `TIME`, `DATETIME`, `TIMESTAMP`, `YEAR` |
| Structured | `JSON` |
