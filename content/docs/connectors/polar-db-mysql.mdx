## Before you begin

### Source

<SourceModeTabs label="Choose the PolarDB for MySQL source read mode">
<SourceModeTab value="snapshot">

Use **Snapshot** to copy the rows that exist now. Allow the TapState runtime network through the PolarDB cluster whitelist, then grant a reader access to the selected database:

```sql
CREATE USER 'tapstate_reader'@'%' IDENTIFIED BY 'replace-with-a-secret';
GRANT SELECT, SHOW VIEW ON `orders`.* TO 'tapstate_reader'@'%';
FLUSH PRIVILEGES;
```

</SourceModeTab>
<SourceModeTab value="snapshot-cdc">

Use **Snapshot + CDC** to copy existing rows and continue from MySQL Binlog changes.

```sql
GRANT SELECT, SHOW VIEW ON `orders`.* TO 'tapstate_reader'@'%';
GRANT REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'tapstate_reader'@'%';
FLUSH PRIVILEGES;
```

Enable Binary Logging for the cluster and retain logs long enough to cover the longest planned interruption. A seven-day retention window is a useful starting point, but size it for the actual change rate and recovery objective.

</SourceModeTab>
</SourceModeTabs>

### Target

Use a dedicated writer for destination tables:

```sql
CREATE USER 'tapstate_writer'@'%' IDENTIFIED BY 'replace-with-a-secret';
GRANT SELECT, INSERT, UPDATE, DELETE ON `analytics`.* TO 'tapstate_writer'@'%';
GRANT CREATE, ALTER, DROP ON `analytics`.* TO 'tapstate_writer'@'%';
FLUSH PRIVILEGES;
```

## Create a connection

```yaml title="tapstate-work/source/orders_polardb.tapstate.yml"
version: tapstate/v1
kind: source
id: orders_polardb
connector: polar-db-mysql
config:
  host: pc-abc.mysql.polardb.rds.aliyuncs.com
  port: "3306"
  database: orders
  username: ${POLARDB_READER_USER}
  password: ${POLARDB_READER_PASSWORD}
  addtionalString: useUnicode=yes&characterEncoding=UTF-8
  timezone: "+00:00"
mode: cdc
tables:
  - orders
```

Use the primary cluster endpoint appropriate for the selected role. For a target, omit `mode` and `tables` and provide writer credentials.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

<ValidationStatusGuide />

Stage a snapshot and a short CDC run before production. Verify cluster access from the runtime network and inspect Binlog retention after a controlled interruption.

## Limitations

- CDC needs Binary Logging and retained Binlog files; if the capture position expires, re-snapshot the affected tables.
- Whitelist, endpoint, and account configuration are enforced by Alibaba Cloud and PolarDB at runtime.
- Test target table evolution and conflict behavior with representative data before enabling production writes.

## Reference

| Field | Required | Default | Use |
|---|---:|---|---|
| `host` | Yes | — | PolarDB cluster endpoint. |
| `port` | Yes | — | MySQL listener port. |
| `database` | Yes | — | Database to connect to. |
| `username`, `password` | Username yes | — | PolarDB account credentials. |
| `addtionalString` | No | `useUnicode=yes&characterEncoding=UTF-8` | Additional JDBC parameters; retain the inherited spelling. |
| `timezone` | No | Empty | UTC offset for temporal conversion. |
