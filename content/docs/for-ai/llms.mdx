TapState publishes reader HTML and AI-readable Markdown from the same source tree. The [llms.txt proposal](https://llmstxt.org/) provides a compact discovery layer, but it does not replace canonical HTML, page metadata, or the sitemap.

## Discovery surfaces

Use the smallest surface that answers the task:

| Surface | URL | Purpose |
|---|---|---|
| Canonical documentation | `https://tapstate.com/docs` | Primary human and crawler surface with canonical metadata and structured data. |
| `llms.txt` | `https://tapstate.com/llms.txt` | Compact index with page links and descriptions. |
| `llms-full.txt` | `https://tapstate.com/llms-full.txt` | Combined documentation context for tools that need a single file. |
| Page Markdown | `https://tapstate.com/llms.mdx/docs/<page>/content.md` | Canonical page body plus explicit agent metadata and expanded tabs. |
| Sitemap | `https://tapstate.com/sitemap.xml` | Canonical reader-page discovery for search engines and crawlers. |

## Give an assistant context

For product discovery, start with `llms.txt`. For a connector or DSL decision, provide the page-level Markdown so the assistant receives the exact maturity, TapState availability, modes, limitations, and every tab body.

```text
Read https://tapstate.com/llms.mdx/docs/connectors/mysql/content.md,
then help me prepare a MySQL Snapshot + CDC source.
```

The public site does not currently expose standalone JSON Schema or example-corpus URLs. Use the DSL reference and current CLI validation until those routes are published and verified.

## Content contract

- Reader HTML is canonical and includes canonical URLs, breadcrumb data, and article structured data.
- Connector frontmatter is a compact discovery index; the page body remains the authoritative explanation.
- Interactive tabs expand into labeled sections in Page Markdown and `llms-full.txt`.
- Maturity and TapState availability are separate signals. Do not infer current product availability from upstream connector maturity.
- Unsupported UI, runtime, TLS, and connection-test behavior is not inferred from upstream documentation.

## MCP integration

<Aside type="note" title="Planned for a later runtime phase">
  The current release supports offline CLI authoring and validation. It does not expose a live MCP control endpoint.
</Aside>

When a future TapState runtime exposes MCP, an AI agent will be able to operate it through the published protocol endpoint. See [MCP Integration](/docs/for-ai/mcp) for the planned boundary.
