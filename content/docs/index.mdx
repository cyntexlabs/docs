# Documentation

**Kafka moves events. Warehouses analyze history. TapState turns database truth into live operational state.**

TapState captures changes from operational databases, transforms them in flight, and serves continuously fresh state through one deployable data path — replacing the CDC tool + broker + stream processor + cache assembly project. The current open-source release ships the offline authoring layer: create, inspect, and validate declarative `.tapstate.yml` resources with deterministic guardrails ([release status](/docs/overview/release-status)).

## Why TapState

- **One declarative contract:** sources, pipelines, transforms, and serving intent are ordinary, reviewable files that work with Git, editors, and automation.
- **Guardrails before deployment:** structure, references, connector fields, and capability combinations are checked offline, with stable diagnostic codes and suggested fixes.
- **Built for people and agents:** deterministic scaffolding, JSON/YAML output, JSON Schema, and an embedded connector catalog remove guesswork for humans and AI assistants alike.
- **One product boundary instead of an assembly stack:** the target architecture unifies capture, transformation, and queryable operational state.
