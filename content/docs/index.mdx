# TapState Docs

TapState is being built to capture operational changes, transform them in flight, and serve continuously fresh state through one deployable data path. The current open-source release delivers the offline authoring layer: create, inspect, and validate declarative `.cyn.yml` resources without a server or database connection.

## Why TapState

The current CLI already establishes several foundations for the broader product:

- **One declarative contract:** sources, pipelines, transforms, views, and serving intent are ordinary, reviewable files.
- **Guardrails before deployment:** structure, references, connector fields, and capability combinations can be checked offline.
- **Built for people and agents:** deterministic scaffolding, JSON/YAML output, coded diagnostics, JSON Schema, and an embedded connector catalog reduce guesswork.
- **A path beyond the assembly stack:** the target architecture brings capture, transformation, and queryable operational state into one product boundary.
