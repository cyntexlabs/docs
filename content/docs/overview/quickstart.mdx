This quickstart exercises the current release: create declarative resources, inspect them, and validate the workspace offline.

<Aside type="note" title="Offline authoring">
  These steps do not connect to a database or move data — that arrives with the runtime. See [Release Status](/docs/overview/release-status).
</Aside>

## 1. Get the CLI

If a `tapstate` binary is already installed, verify it:

```bash
tapstate --version
```

A hosted installer (`install.tapstate.dev`) is planned. Until it ships, build the native binary from a source checkout with Maven and GraalVM for JDK 21:

```bash
cd /path/to/tapstate-source
mvn -Pnative -pl cli -am -DskipTests package
export PATH="$PWD/cli/target:$PATH"
```

## 2. Create a MySQL source

The non-interactive command is suitable for terminals, scripts, and AI agents:

```bash
tapstate new --non-interactive \
  --kind source \
  --connector mysql \
  --id orders_source \
  --mode snapshot \
  --set host=db.internal \
  --set port=3306 \
  --set database=orders \
  --set username='${MYSQL_USER}' \
  --set password='${MYSQL_PASSWORD}' \
  --workdir tapstate-work
```

The CLI creates `tapstate-work/source/orders_source.tapstate.yml` with canonical `tapstate/v1` YAML. See the [MySQL connector guide](/docs/connectors/mysql) before choosing CDC.

## 3. Create a target connection

A mode-less source resource acts as a target connection supplier:

```bash
tapstate new --non-interactive \
  --kind source \
  --connector mongodb \
  --id customer_view \
  --set isUri=true \
  --set uri='${MONGODB_URI}' \
  --workdir tapstate-work
```

## 4. Connect them with a pipeline

```bash
tapstate new --non-interactive \
  --kind pipeline \
  --id orders_to_customer_view \
  --source orders_source \
  --sync-to customer_view \
  --workdir tapstate-work
```

## 5. Validate the workspace

```bash
tapstate validate --workdir tapstate-work
```

A valid three-resource workspace returns:

```text
valid: 3 resources in tapstate-work
```

Exit code `0` means the offline contract passed: structure, references, connector fields, and mode combinations are all consistent. Credentials, network access, and data movement are verified by the runtime, not by offline validation.

## 6. Inspect and explain

```bash
tapstate ls --workdir tapstate-work
tapstate desc orders_to_customer_view --workdir tapstate-work
tapstate explain source.mode
```

Use `ls` and `desc` for concrete workspace resources. Use `explain` for grammar fields. Add `--output json` when an agent or CI job needs structured results.
