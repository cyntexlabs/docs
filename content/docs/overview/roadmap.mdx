TapState is being developed toward a complete operational-data path. The sequence below describes durable customer outcomes, not dates, release phases, or availability commitments.

<Aside type="note" title="Check release evidence separately">
  A capability belongs to a release only after it appears in versioned artifacts and release notes. See [Release status](/docs/overview/release-status) for the evidence model used by this site.
</Aside>

## Author safely

Define connections and pipelines as reviewable resources. Provide deterministic schema validation, connector-aware fields, closed references, coded diagnostics, and AI-readable documentation.

**Outcome:** teams and assistants can describe the intended data path without guessing its structure.

## Connect and run

Load connector implementations, discover source objects, execute bounded reads and continuous capture, apply transformations, write destinations, and persist recovery positions.

**Outcome:** an authored data path becomes an executable path with explicit lifecycle and recovery behavior.

## Operate reliably

Add status, logs, metrics, lag, controlled lifecycle operations, distributed coordination, schema-change handling, and end-to-end test evidence.

**Outcome:** operators can understand, recover, and change a running path without reconstructing state across unrelated systems.

## Serve governed state

Expose current operational state and approved event outputs through documented interfaces, access controls, lineage, and deployment options.

**Outcome:** applications and agents consume useful state through a governed product boundary.

## Release principles

- Keep the resource model reviewable and deterministic.
- Keep model choice outside TapState; AI clients use published interfaces.
- Require explicit authorization for writes and lifecycle changes.
- Separate product claims from target architecture.
- Publish performance and reliability only with reproducible evidence.
- Preserve compatibility identifiers until their implementation contracts change.
