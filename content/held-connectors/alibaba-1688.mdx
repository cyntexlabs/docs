## Before you begin

### Source

1. Create or select an Alibaba 1688 Open Platform developer application authorized for the marketplace data you need.
2. Complete the platform authorization flow and obtain the authorization value required by your deployment.
3. Confirm the API permissions, rate limits, and accessible business resources with the application owner.

## Create a connection

```bash
tapstate new --non-interactive \
  --kind source \
  --connector ali1688 \
  --id alibaba_orders \
  --mode api \
  --set code='${ALI1688_AUTHORIZATION_CODE}' \
  --workdir tapstate-work
```

```yaml title="tapstate-work/source/alibaba_orders.tapstate.yml"
version: tapstate/v1
kind: source
id: alibaba_orders
connector: ali1688
config:
  code: ${ALI1688_AUTHORIZATION_CODE}
mode: api
```

Use the value accepted by your TapState deployment's Alibaba 1688 authorization flow. Do not substitute App Key or App Secret fields; this connection accepts the documented `code` field.

## Validate the configuration

```bash
tapstate validate --workdir tapstate-work
```

## Limitations

- This connector reads 1688 API data only. It does not write records or apply schema changes.
- Alibaba 1688 enforces authorization scopes, code expiry, API quotas, and marketplace resource access.
- Keep the authorization value outside committed workspace files by supplying it through an environment variable.

## Reference

| Field | Required | Use |
|---|---|---|
| `code` | Yes | Authorization value accepted by the deployment's Alibaba 1688 connector flow. |
